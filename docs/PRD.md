# Immersive AI Reader — 产品需求文档（PRD）

> **文档版本**：v2.2
> **更新日期**：2026-03-25
> **核心定位**：沉浸式 AI 阅读 + 碎片笔记 → 结构化知识的闭环工具
> **目标用户**：研究者、知识分子、终身学习者

---

## 一、产品愿景

让阅读成为知识的入口，而不是终点。

现有的阅读工具要么只读不记（纸质书、Kindle），要么记了找不到（Notion、Obsidian 堆积如山），要么 AI 和笔记割裂（ChatPDF 只读，Notion 只记）。

**我们的差异化定位**：

> 每一条笔记都携带来源锚点（精确的 PDF 位置 + 阅读时间 + 当时的 AI 对话），形成可溯源的知识图谱。三个月后打开一条笔记，能立刻回到「当时为什么这么想」——这是任何 PKM 工具都做不到的。

---

## 二、核心产品闭环

```
┌─────────────────────────────────────────────────────────────┐
│                                                             │
│   阅读捕获 ──→ AI 整理 ──→ 知识产出 ──→ 外部同步           │
│       ↑                                        ↓            │
│       └───────────── 知识回流 ───────────────────┘           │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

**飞轮逻辑**：
- 用户持续阅读 → 捕获层积累碎片笔记
- AI 发现跨文档关联 → 推荐合成
- 用户知识库增长 → 新文档进来时 AI 连接已有知识
- 越读越有深度，形成个人知识护城河

## 三、用户完整旅程

### 旅程 1：刚打开文档 → 进入阅读（两种模式）

**两种阅读模式**：

**Free Mode（自由模式）**
- 侧边栏可用，AI 面板展开，所有工具可见
- 适合：浏览文档、查找资料、多任务处理
- 入口：文档打开后默认进入此模式

**Focus Mode（专注模式）**
- 侧边栏完全收起，AI 面板收为窄条（仅留悬浮按钮）
- 只保留 PDF 阅读区 + 极简工具条
- 适合：深度阅读、知识捕获、沉浸思考
- 入口：点击工具条「专注模式」按钮，或使用快捷键 `Cmd+Shift+F`
- 退出：`Escape` 键或点击「退出专注」按钮

**模式切换的价值**：
- Focus Mode 是一种「认知仪式」——用户主动宣告「我要深度阅读了」
- 心理学证明：明确的开始/结束标记能提升专注质量和记忆留存
- Free Mode 保证日常浏览不会因为「太正式」而造成使用摩擦

**进度恢复**：
- Focus Mode 退出时，自动保存：当前页码、滚动位置、AI 会话上下文
- 再次进入同一文档的 Focus Mode 时，自动恢复到上次退出的位置
- 如果中途切换到 Free Mode 或关闭文档，进度仍然保留（以页码为单位）

**实现要点**：
- 工具条包含：缩放、页码、搜索、「专注模式」切换按钮、一个「AI 助手」悬浮按钮
- Focus Mode 下，悬浮按钮仍然可用，但不展开面板（点击后弹出 mini AI 交互窗口，不跳转上下文）

**产品状态**：
- `activeTabId === 'library'` 时不加载 canvas，避免在 Library Tab 报错 ✅
- AI Panel 已可折叠收起 ✅

### 旅程 2：阅读中，边读边思考 → 最小打断捕获

**三个层次的捕获设计**（仅在 Focus Mode 下激活）：

#### L1 自动高亮（零打断）
- **触发**：用户选中文本（mouseup / touchend 检测到选中文字）
- **表现**：选中区域出现蓝色薄层高亮（rgba(59, 130, 246, 0.2)），150ms 淡入动画
- **气泡**：右侧弹出单个「加入笔记」按钮气泡，2 秒后自动消失
- **后台记录**：`{ pageNumber, originalText, timestamp, sessionId }`
- **防抖**：1 秒内重复选同一区域不重复记录
- **边界**：选区跨页 → 只高亮当前页部分，气泡提示「跨页选区已截断」

#### L2 即时 AI 交互（最小打断，直接展开选项）

**触发**：选中文字后，弹出浮层直接显示四个选项（不需要点击「AI」按钮）：

```
┌──────────────────────────┐
│ 🤖 让 AI 解释             │
│ 🌐 让 AI 翻译             │
│ 💬 加入 AI 会话           │
│ 📝 新建笔记               │
└──────────────────────────┘
```

- **让 AI 解释**：紧贴选区弹出浮层，AI 回复流式显示，完成后底部出现「保存到笔记」或「关闭」
- **让 AI 翻译**：左右分栏，左原文右译文，流式输出，完成后可「保存原文+译文」或「仅保存译文」
- **加入 AI 会话**：选区追加到 AI Panel 输入框，光标跳转，AI Panel 自动展开（Focus Mode 下展开为 mini 窗口）
- **新建笔记**：紧贴高亮弹出 mini 编辑器，自动带入原文引用，用户输入后保存

**防错设计**：
- AI 服务不可用 → 显示「AI 服务暂时不可用，请检查网络设置」，不崩溃
- 选区为空时 → 浮层不出现
- 快速连续点击 → debounce 300ms，防止重复请求
- 用户不操作 → 10 秒后浮层自动消失（可配置）

#### L3 深度笔记（专注输入）

**触发**：双击已有高亮块

**编辑器表现**：
- 全宽浮层，覆盖 PDF 上半部分（高亮本身始终可见）
- 顶部显示：「第 3 页 · 来自高亮」（来源锚点预览）
- 支持 Markdown，底部有标签输入
- 点击高亮块外部 → 弹出「是否放弃未保存笔记？」确认框

#### 捕获面板入口（Focus Mode 下）

- Focus Mode 工具条左侧有一个「捕获记录」悬浮按钮（📋 图标）
- 点击展开捕获面板抽屉，从左侧滑入，不遮挡 PDF 核心区域
- Focus Mode 下，捕获面板保持收起状态（减少干扰）

**产品能力**：
- 右键菜单 + 浮出工具条（`onAddNoteSelection` 已在 MainCanvas 实现）
- 每条笔记存储锚点：`pageNumber + x + y + width + height + 原文`

### 旅程 3：AI 对话中 → 结果可沉淀

**AI 回复引用可点击**：
- AI 回复中的 `[ref:p3]` 格式可点击跳转，跳转到页面后高亮被引用文字，2 秒后淡出到半透明

**每条 AI 回复的「保存到笔记」按钮**：
- AI 回复完成后，底部自动出现：`💾 保存到笔记`
- 已保存过一次显示：「已保存 ✓」（灰色，点击可重新编辑）
- 保存时自动关联 citedPages，建立来源锚点

**对话历史导出**：
- AI Panel 右上角菜单，支持导出为：
  - Markdown（带页码引用）
  - 纯文本
  - 带结构的 Markdown（按会话分节）

### 旅程 4：读完整篇文档 → 给用户一个「收获了什么」的清晰答案

**触发条件**（首次满足时提示）：
- 阅读进度 ≥ 80% **且** 捕获数量（高亮 + 笔记）≥ 3

**提示方式**（右下角小卡片，不打断阅读）：

```
┌─────────────────────────────────────┐
│ 📖 阅读完成！你积累了 7 条标注        │
│                                      │
│  是否生成阅读摘要？                   │
│                                      │
│  [生成摘要]  [稍后]  [不需要]        │
└─────────────────────────────────────┘
```

- **生成摘要** → 弹出 mini AI 窗口生成摘要，完成后追加到捕获面板
- **稍后** → 关闭，3 小时后或下次打开同一文档再次提示
- **不需要** → 关闭，今天不再提示

**AI 摘要卡片内容**：

```markdown
## 📄 阅读摘要 · {文档标题}

### 核心观点
1. ...
2. ...

### 你标注最多的主题
- 注意力机制（4 处）
- ...

### 相关已有知识
你在《深度学习》文档中也标注过「注意力」，是否关联？
→ [创建跨文档笔记]
```

**关联已有知识**：是知识飞轮的核心触发点，让用户感受到知识在增长。

### 旅程 5：碎片笔记整理成结构化知识

**Focus Mode 下的捕获面板**：
- Focus Mode 工具条左侧有「捕获记录」按钮（📋 图标），点击从左侧滑出抽屉
- 退出 Focus Mode 后，捕获面板收起

**Free Mode 下的捕获面板**：
- 作为左侧栏独立 Tab（与 Library、Settings 并列）
- 常驻可用，完整功能

**时间线布局**（不是列表视图）：

```
2024-03-20 14:32
  ┌─ 📝 笔记：Self-attention 的优势在于…
  │    来源：p.3 · 来自高亮
  │
  └─ 💬 AI 回复：Transformer 的并行计算优势…
       来源：p.3 · AI 对话

2024-03-20 14:28
  └─ 🔵 高亮：「完全基于注意力机制」
       来源：p.3
```

**筛选和搜索**：
- 顶部：全部 / 仅笔记 / 仅高亮 / 仅 AI 对话
- 全文搜索（在笔记内容中高亮匹配）

**「一键合成」按钮**（捕获面板顶部，蓝色主按钮）：
- **前置条件**：至少 3 条捕获
- 点击后 AI Panel 展开，流式输出结构化 Markdown
- 结果可编辑、可拆分、可合并到已有笔记
- 入口二：文档读完提示卡片中也有「合成笔记」按钮

**合成 Prompt 逻辑**：
```
基于以下捕获内容，生成一份结构化笔记：

[高亮1 - p.3] ...
[笔记1 - p.3] ...
[AI回复1 - p.3] ...

要求：
- 提炼 3-5 个核心观点
- 用 Markdown 格式
- 保留每条引用的来源页码
- 不要重复，整合归纳
```

**笔记层级**：
- 闪念笔记：来自 L1 自动高亮、L2 AI 即时回复，原始未整理
- 永久笔记：用户主动整理或 L3 深度笔记，结构化带标签
- 项目笔记：多个永久笔记合并，跨文档有明确主题

### 旅程 6：知识检索和复用 → 三个月后找到这条笔记

**搜索结果展示**：

```
📄 Attention Is All You Need · p.3
"Self-attention 的优势在于…"
来源：Attention Is All You Need.pdf
阅读时间：2024-03-20 14:32

"注意力机制分为…"
```

每条结果显示：**文档名 + 页码 + 阅读时间**——三个月后看到这些元数据，能瞬间唤回记忆。

**点击跳转流程**：
1. 打开对应 PDF
2. 跳转到对应页
3. 高亮对应文本位置，高亮保持 3 秒可见后淡出到半透明

**笔记间关系**（进阶功能，P2）：
- 笔记之间有引用关系时，在捕获面板显示知识图谱节点连线
- 数据结构预留：笔记 `relatedNotes[]` 字段

### 旅程 7：知识同步到外部 PKM 系统

**导出粒度**：单条笔记 / 单文档全部 / 按标签批量

**Obsidian 导出格式**（每条笔记一个 `.md` 文件）：

```markdown
---
uid: 20240320-143200-attention
source: Attention Is All You Need.pdf
page: 3
tags: [AI, Transformer, 注意力机制]
created: 2024-03-20
---

# Self-attention 的优势

原文引用：
> "完全基于注意力机制" — Attention Is All You Need.pdf p.3

笔记正文：
Self-attention 让序列内任意位置可以直接建立关联，
解决了 RNN 的长距离依赖问题。
```

文件命名：`{date}-{first-tag}-{slug}.md`

**Vault 路径**：用户首次设置时通过文件夹选择器配置，记住选择可随时修改

**双向同步**（P2 进阶功能）：MVP 阶段只做单向导出（Reader → Obsidian），双向同步后续版本再做

## 六、核心数据模型

### 笔记锚点（Source Anchor）

每条笔记携带来源锚点，点击可回溯：

```typescript
interface SourceAnchor {
  documentId: string;
  documentTitle: string;       // "Attention Is All You Need.pdf"
  pageNumber: number;         // 3
  textRange: {
    x: number;                // 左上角 x（CSS px）
    y: number;                // 左上角 y（CSS px）
    width: number;
    height: number;
  };
  originalText: string;       // 高亮的原文（用于验证和回显）
  readingSessionId: string;    // 本次阅读 session ID
  readAt: Date;               // 阅读时间
}
```

### 笔记层级（Note Hierarchy）

```
闪念笔记 (Fleeting Note)
  └─ 来自：L1 自动高亮、L2 AI 即时回复
  └─ 特征：原始、未整理、可丢弃

永久笔记 (Permanent Note)
  └─ 来自：用户主动整理闪念笔记 / L3 深度笔记
  └─ 特征：结构化、带标签、可引用

项目笔记 (Project Note)
  └─ 来自：多个永久笔记合并
  └─ 特征：跨文档、有明确主题、可导出
```

### 笔记存储结构

```typescript
interface Annotation {
  id: string;
  documentId: string;
  type: 'highlight' | 'note' | 'ai-card';

  // 文本内容
  text: string;  // __NOTE__|content\n\n[引用原文]

  // 位置
  positionX: number;
  positionY: number;
  pageNumber: number;

  // 来源锚点（核心差异化字段）
  sourceAnchor?: SourceAnchor;

  // 元数据
  tags: string[];
  sessionId?: string;
  createdAt: Date;
  updatedAt: Date;
}
```

## 七、已确认的产品决策

### Q1: 来源锚点的存储精度

**已确认：方案 C（混合）**
- 默认方案 A（pageNumber + x/y + originalText），保证 MVP 可用
- 用户可开启「精确锚点」，存储 PDF 截图快照（~50KB/条）
- 技术上：先实现方案 A，方案 B 作为可选升级

### Q2: AI 对话自动存笔记

**已确认：方案 C（捕获面板 → 知识库）**
- AI 回复默认存入捕获面板（可丢弃，零摩擦）
- 用户主动操作「升级到知识库」，有筛选感
- 这是 Zettelkasten 的核心哲学：先捕获，后整理

### Q3: 合成笔记的触发时机

**已确认：方案 B（自动提示 + 用户确认）**
- 阅读进度 ≥ 80% 且捕获数 ≥ 3 时，首次满足时提示
- 用户点「生成」→ AI 产出摘要卡片追加到捕获面板
- 用户点「稍后」→ 3 小时后或下次打开再次提示
- 用户点「不需要」→ 当天不再提示
- **禁止**：自动生成并直接进入知识库——用户需要掌控感

### Q4: 跨文档关联推荐的触发条件

**已确认：方案 C（递进触发）**
- L1：标签匹配先做（实现简单）
- L2：AI 语义聚类后做（需要 AI 能力，复杂度高）

### Q5: 知识卡片的最小信息单元

**已确认：标配**
```typescript
interface MinimalNote {
  content: string;        // 用户笔记正文
  anchor: SourceAnchor;    // pageNumber + documentId + originalText
  createdAt: Date;
}
```
永久笔记额外包含：`tags`, `updatedAt`

### Q6: Obsidian vault 路径

**已确认：方案 A + 提供默认值**
- 首次设置引导用户选择 vault 路径（文件夹选择器）
- 记住用户选择，可在设置中随时修改

### Q7: Focus Mode（专注阅读模式）

**已确认：独立模式 + 进度完整恢复**
- Focus Mode：侧边栏收起，AI 面板收窄，只留 PDF + 极简工具条
- Free Mode：全部面板可用，适合浏览和多任务
- 切换方式：工具条按钮或 `Cmd+Shift+F`
- 退出时保存进度，再次进入自动恢复：
  - 当前页码 + scrollTop
  - AI 会话上下文（对话历史 + AI Panel 展开/收起状态）
- 不做全屏独占（允许切换，不强制）

**L2/L3 在两种模式下均可用**：
- Focus Mode：L1 自动高亮辅助感知，UI 更紧凑（浮层 200px、动画 80ms）
- Free Mode：L2 直接触发（无需点击气泡），浮层稍大（220px），动画 120ms
- 两种模式下的 L3 深度笔记编辑器行为完全一致

**再次进入提示**：
- 检测到「上次在 Focus Mode 中阅读过」时，显示：「上次你读到这里，要继续专注阅读吗？」
- 点「继续」→ 恢复到上次页码 + scrollTop + AI 会话状态

### Q8: AI 标签推荐

**已确认：默认关闭（可手动开启）**
- AI 速度不稳定，MVP 阶段默认关闭
- 用户可在 Settings 中开启「AI 标签推荐」
- 开启后，仅在以下时机推荐：保存笔记时 + AI 生成摘要时
- 不在任何其他时机弹标签推荐，避免干扰

### Q9: Focus Mode mini AI 窗口

**已确认：右侧滑出，窄屏底部抽屉**
- 默认从右侧滑出（宽度 320px），对 PDF 遮挡最小
- 屏幕宽度 < 900px 时，改为底部抽屉模式
- 不自动展开，用户主动点击悬浮按钮触发
- 关闭只收起窗口，不切换模式

### Q10: 捕获面板时间线

**已确认：日期 + 时间双重分组**
- 按日期一级分组（2024-03-20）
- 日期内按时间二级分组（14:32）
- 同时间可有多个捕获项并列

### Q11: 合成笔记发布后

**已确认：原始捕获保留**
- 合成笔记作为新的独立 capture 存入
- 原始捕获全部保留，并列显示
- 合成笔记带「✨ 已合成」标签，位于捕获面板顶部
