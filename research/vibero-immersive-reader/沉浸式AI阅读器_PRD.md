## 1. 产品概述

### 1.1 产品定位

沉浸式 AI 阅读器是一款面向科研工作者、学生和知识工作者的智能文档阅读工具。通过将静态 PDF 文档转换为交互式 AI 画布，结合大语言模型（LLM）的能力，提供非线性、上下文感知的阅读体验。

### 1.2 核心价值主张

- **打破线性阅读限制**：从静态 PDF 到交互式画布，支持非线性探索
- **AI 深度理解**：基于文档上下文的智能问答和内容解析
- **代码与论文协同**：在同一工作区理解论文原理与代码实现
- **智能知识管理**：自动提取关键点，生成可交互的笔记和标注

### 1.3 产品愿景

让每一次阅读都成为深度理解和知识创造的旅程，通过 AI 技术重新定义人与文档的交互方式。

## 2. 市场分析

### 2.1 目标市场

- **学术研究人员**：需要频繁阅读和分析学术论文的科研工作者
- **研究生/博士生**：进行文献综述和深度学习的学术群体
- **技术开发者**：需要理解论文原理并实现代码的工程师
- **知识工作者**：需要快速掌握新领域知识的专业人士

### 2.2 市场痛点

1. **信息过载**：学术论文数量爆炸，难以高效筛选和理解
2. **线性阅读低效**：传统 PDF 阅读器无法满足跳跃式探索需求
3. **理论与实践脱节**：论文理解与代码实现之间存在鸿沟
4. **知识碎片化**：阅读笔记分散，难以形成系统性知识体系
5. **语言障碍**：非母语论文理解困难，需要频繁查词翻译

### 2.3 竞品分析

| 产品 | 核心功能 | 优势 | 劣势 |
|------|---------|------|------|
| **Vibero.dev** | 交互式画布、论文+代码协同、AI 对话 | 非线性阅读体验好，上下文理解强 | 价格较高，主要面向论文阅读 |
| **ChatPDF** | PDF 文档对话、问答 | 简单易用，价格亲民 | 缺乏画布交互，功能单一 |
| **Paperpal** | 学术写作、文献管理、润色 | 功能全面，写作辅助强 | 阅读体验一般，侧重写作 |
| **Semantic Scholar** | AI 论文搜索、推荐 | 论文库丰富，推荐精准 | 缺乏深度阅读工具 |
| **Microsoft Immersive Reader** | 沉浸式阅读、朗读 | 辅助功能强，免费 | 无 AI 对话，无画布功能 |

### 2.4 差异化优势

1. **画布式交互**：类似 Vibero 的无限画布，支持拖拽、缩放、自由布局
2. **多模态协同**：论文文本 + 代码仓库 + AI 对话三位一体
3. **上下文感知**：AI 理解当前阅读位置和标注内容，提供精准回答
4. **知识图谱化**：自动提取实体、概念，构建可视化知识网络

### 4.2 详细功能需求

#### 4.2.1 交互式画布（核心功能）

**功能描述**：
将 PDF 文档转换为可在无限画布上自由布局的交互式内容块。

**用户交互流程**：

1. **文档上传**
   - 用户拖拽或点击上传 PDF 文件
   - 系统解析文档结构（标题、章节、图表、公式）
   - 自动将文档拆分为可独立交互的内容块

2. **画布布局**
   - 文档内容块自动排列在画布上
   - 用户可自由拖拽、缩放、重新排列
   - 支持放大查看细节，缩小查看全局
   - 支持创建多个视图区域

3. **标注与高亮**
   - 选中文本进行高亮（多种颜色）
   - 添加文本标注和评论
   - 绘制手绘标记
   - 标注自动关联到原文位置

4. **笔记卡片**
   - 在画布上创建笔记卡片
   - 支持富文本编辑（Markdown）
   - 可关联到文档特定位置
   - 支持拖拽重新布局

**交互细节**：

| 操作 | 交互方式 | 响应 |
|------|---------|------|
| 上传文档 | 拖拽文件到画布 / 点击上传按钮 | 显示上传进度，解析后自动展开 |
| 缩放画布 | 鼠标滚轮 / 双指捏合 | 平滑缩放，显示缩放比例 |
| 拖拽内容块 | 点击并拖动 | 实时跟随，吸附对齐 |
| 高亮文本 | 选中文本 → 选择颜色 | 即时高亮，显示标注工具栏 |
| 创建笔记 | 双击空白处 / 快捷键 | 创建笔记卡片，自动聚焦 |

**技术要求**：
- 画布渲染：使用 Canvas API 或 WebGL
- 文档解析：PDF.js + 自定义解析逻辑
- 布局引擎：类似 Figma 的自动布局算法
- 性能优化：虚拟滚动、懒加载、缓存机制

---

#### 4.2.2 AI 智能助手（核心功能）

**功能描述**：
基于文档上下文的 AI 对话助手，支持问答、解释、摘要、翻译等功能。

**用户交互流程**：

1. **上下文对话**
   - 用户点击或选中画布上的内容块
   - AI 助手自动获取该内容块的上下文
   - 用户提问，AI 基于上下文回答
   - 对话历史保存在侧边栏

2. **智能问答**
   - 用户输入问题（文本或语音）
   - AI 检索文档相关部分
   - 生成答案并标注引用来源
   - 支持追问和多轮对话

3. **概念解释**
   - 用户点击专业术语
   - AI 提供术语解释、背景知识、相关论文
   - 支持展开查看详细说明

4. **摘要生成**
   - 一键生成文档摘要
   - 支持不同粒度（段落级、章节级、全文级）
   - 可自定义摘要长度和重点

5. **翻译功能**
   - 选中段落即时翻译
   - 支持多语言互译
   - 保留专业术语原文
   - 双语对照显示

**交互细节**：

| 操作 | 交互方式 | 响应 |
|------|---------|------|
| 选中内容提问 | 选中 → 右键"询问 AI" | 打开对话窗口，自动关联上下文 |
| 拖拽问答到画布 | 拖动对话气泡到画布 | 创建笔记卡片，保存问答内容 |
| 查看引用来源 | 点击答案中的引用链接 | 高亮并滚动到原文位置 |
| 语音输入 | 点击麦克风图标 | 开始录音，转文字后发送 |

**AI 能力要求**：
- 模型：支持 GPT-4、Claude、Gemini 等主流模型
- 上下文窗口：至少 32K tokens
- RAG（检索增强生成）：基于文档内容的精准检索
- 引用溯源：答案必须标注原文出处

---

#### 4.2.3 论文与代码协同

**功能描述**：
在同一工作区同时查看论文和 GitHub 代码仓库，建立论文概念与代码实现的关联。

**用户交互流程**：

1. **代码仓库集成**
   - 用户输入 GitHub 仓库 URL
   - 系统克隆并解析代码结构
   - 在画布上显示代码文件树

2. **论文-代码关联**
   - AI 自动识别论文中提到的算法/方法
   - 在代码中查找对应实现
   - 建立双向链接（论文段落 ↔ 代码函数）

3. **代码预览与跳转**
   - 点击论文中的算法名称
   - 自动打开对应的代码文件
   - 高亮关键代码行
   - 支持代码语法高亮

4. **代码对话**
   - 用户可针对代码提问
   - AI 解释代码逻辑
   - 支持代码运行结果预览（可选）

**交互细节**：

| 操作 | 交互方式 | 响应 |
|------|---------|------|
| 添加代码仓库 | 输入 GitHub URL / 粘贴代码 | 克隆并解析，显示文件树 |
| 跳转到代码 | 点击论文中的代码引用 | 打开代码面板，定位到函数 |
| 关联论文与代码 | 选中论文段落 → 拖拽到代码 | 创建关联链接 |
| 询问代码 | 右键代码 → "解释这段代码" | AI 生成代码说明 |

**技术要求**：
- Git 集成：使用 isomorphic-git 或类似库
- 代码解析：AST 分析，支持主流语言
- 关联算法：基于文本相似度 + 结构化匹配

---

#### 4.2.4 知识管理

**功能描述**：
自动提取文档关键信息，构建个人知识库和知识图谱。

**用户交互流程**：

1. **自动提取关键点**
   - AI 分析文档结构和内容
   - 提取关键概念、方法、结论
   - 生成关键点列表

2. **实体识别与链接**
   - 识别人名、机构、方法名、数据集等实体
   - 自动链接到外部知识库（Wikipedia, DBLP）
   - 支持用户手动编辑和补充

3. **知识图谱可视化**
   - 以节点图形式展示概念关系
   - 支持拖拽、缩放、筛选
   - 点击节点查看详细信息

4. **导出与分享**
   - 导出为 Markdown、PDF、图片
   - 生成分享链接
   - 支持协作编辑（未来版本）

**交互细节**：

| 操作 | 交互方式 | 响应 |
|------|---------|------|
| 查看关键点 | 点击"关键点"按钮 | 在侧边栏显示列表 |
| 查看知识图谱 | 点击"知识图谱"按钮 | 全屏显示图谱视图 |
| 导出笔记 | 点击"导出" → 选择格式 | 下载文件 |

### 4.3 非功能性需求

#### 4.3.1 性能要求

| 指标 | 目标值 |
|------|--------|
| PDF 解析速度 | < 5 秒（10 页文档） |
| 画布渲染帧率 | ≥ 60 FPS |
| AI 响应时间 | < 3 秒（首字输出） |
| 支持文档大小 | ≤ 100 MB / 篇 |
| 并发用户数 | ≥ 10,000 |

#### 4.3.2 可用性要求

- 响应式设计：支持桌面端（优先）、平板（未来）、移动端（未来）
- 浏览器兼容：Chrome 90+、Firefox 88+、Safari 14+、Edge 90+
- 无障碍支持：符合 WCAG 2.1 AA 标准
- 多语言支持：中文、英文（v1.0），其他语言（未来）

#### 4.3.3 安全性要求

- 数据加密：传输层 TLS 1.3，存储层 AES-256
- 用户隐私：符合 GDPR、CCPA 等法规
- 访问控制：基于角色的权限管理
- 数据备份：每日自动备份，保留 30 天

#### 4.3.4 可扩展性要求

- 微服务架构：支持独立扩展各模块
- 插件系统：支持第三方插件（未来版本）
- API 开放：提供 RESTful API 和 Webhook

## 10. 风险与挑战

### 10.1 技术风险

| 风险 | 影响 | 缓解措施 |
|------|------|---------|
| 大模型 API 成本高 | 利润率低 | 优化 Prompt，缓存常见答案，考虑自部署模型 |
| PDF 解析复杂格式失败 | 用户体验差 | 多引擎备份，人工审核边缘情况 |
| 画布性能瓶颈 | 用户流失 | 虚拟滚动、WebWorker、增量渲染 |

### 10.2 市场风险

| 风险 | 影响 | 缓解措施 |
|------|------|---------|
| 竞品模仿速度快 | 差异化减弱 | 快速迭代，构建数据护城河，社区运营 |
| 用户付费意愿低 | 收入不及预期 | 提供免费版，教育优惠，企业定制 |
| 学术圈接受度慢 | 增长缓慢 | KOL 合作，高校试点，论文发表 |

### 10.3 法律风险

| 风险 | 影响 | 缓解措施 |
|------|------|---------|
| 论文版权问题 | 法律纠纷 | 仅解析公开论文，用户自行上传负责 |
| AI 生成内容准确性 | 用户误导 | 标注"AI 生成"，提供引用来源 |
| 数据隐私合规 | 罚款、下架 | 符合 GDPR、CCPA，数据本地化 |
